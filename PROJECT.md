## 目的

システム全体の構成を俯瞰出来るライブラリを構築する。

## 背景

データ基盤構築の業務を担当することとなった。

主に、以下を構築する。
1. データパイプライン
2. データモデル(ER図など)
3. インフラ

システム全体の構成を俯瞰出来ることが重要となった。

## 機能概要

1. D3.js で俯瞰図を描画する。
2. Next.js のクライアント・コンポーネントで描画する。
3. 描画する図のデータ構造を確立させる。
4. 描画する図は データ と 描画 を分離し、それぞれでクラスを定義する。
5. 描画する図は terraform, er図 など複数あり、それぞれ別フォルダで管理する。(最終的に描画ライブラリに分離できるようにする)
6. 描画する複数の図は一つの svg 内で同時に描画することを前提とする。(terraform, snowflake, dbt, などが併記される)

## 機能詳細

### terraform

1. `.tr` ファイルを読み込み描画用のデータを生成する。 
2. 生成したデータを元にデータクラスを作成する。
3. データクラスを元に描画クラスを作成する。
4. 描画クラスを元に描画する。

### snowflake

1. snowflake の database を取得する。
2. database の オブジェクト を全て取得する。
3. 取得したデータを元にデータクラスを作成する。
3. データクラスを元に描画クラスを作成する。
4. 描画クラスを元に描画する。
